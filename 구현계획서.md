STM32F411CEU6용 베어메탈 C 펌웨어를 구현해줘. HAL은 사용하지 않고 CMSIS 레지스터 직접 접근 방식으로 작성한다.

## 목표

Board1은 로봇팔의 1~4축 스텝모터를 제어한다.  
Raspberry Pi에서 MCP2515 CAN 통신으로 받은 trajectory point를 STM32가 Queue에 저장하고, 1ms 타이머에서 선형 보간한 뒤, 10us 타이머에서 STEP/DIR 펄스를 생성한다.


## CAN 프로토콜

Raspberry Pi에서 STM32로 보내는 제어 패킷은 8바이트이다.

### 대상 CAN ID

| CAN ID | 대상 |
|---|---|
| 0x101 | Board1, 1~4축 |
| 0x102 | Board2, 5축 |
| 0x103 | Board3, 서보보드 |

Board1 펌웨어에서는 0x101만 처리한다.

### 위치 명령 패킷 구조

| Byte | 내용 |
|---|---|
| Byte0 | 상위 4bit Flags, 하위 4bit Motor ID |
| Byte1~4 | Target Position, int32_t, Little Endian |
| Byte5~6 | Speed, uint16_t, Little Endian |
| Byte7 | Duration, 5ms 단위 |

### Byte0 구조

```text
Bit7 Bit6 Bit5 Bit4 | Bit3 Bit2 Bit1 Bit0
Flags                | Motor ID
````

Flags 정의:

| Bit  | 의미            |
| ---- | ------------- |
| Bit7 | Execute       |
| Bit6 | Relative Move |
| Bit5 | Step Mode     |
| Bit4 | Reserved      |

Motor ID:

| Motor ID | 대상 |
| -------- | -- |
| 0        | 1축 |
| 1        | 2축 |
| 2        | 3축 |
| 3        | 4축 |

Board1은 Motor ID 0~3만 허용하고, 그 외 값은 무시

## CAN 언패킹 공식

반드시 아래 방식으로 언패킹한다.

```c
uint8_t motor_id = RxData[0] & 0x0F;
uint8_t flags    = RxData[0] >> 4;

uint8_t execute   = (flags & 0x08) ? 1 : 0;
uint8_t relative  = (flags & 0x04) ? 1 : 0;
uint8_t step_mode = (flags & 0x02) ? 1 : 0;

int32_t target_pos = (int32_t)(
    ((uint32_t)RxData[1]) |
    ((uint32_t)RxData[2] << 8) |
    ((uint32_t)RxData[3] << 16) |
    ((uint32_t)RxData[4] << 24)
);

uint16_t speed = (uint16_t)(
    ((uint16_t)RxData[5]) |
    ((uint16_t)RxData[6] << 8)
);

uint8_t duration_5ms = RxData[7];
```

주의:

* `target_pos`는 음수도 가능하므로 int32_t로 처리한다.
* Byte1~4는 Little Endian이다.
* Byte5~6도 Little Endian이다.
* Duration은 `duration_ms = duration_5ms * 5`로 변환한다.

## CAN 제어 명령

위치 명령과 제어 명령은 분리한다.

| CAN ID | 용도               |
| ------ | ---------------- |
| 0x001  | Emergency Stop   |
| 0x010  | Enable / Disable |
| 0x020  | Homing Start     |
| 0x030  | Clear Error      |
| 0x101  | Board1 위치 명령     |
| 0x201  | Board1 상태 응답     |

### Enable / Disable

CAN ID: 0x010

| Byte  | 내용                    |
| ----- | --------------------- |
| Byte0 | 0: Disable, 1: Enable |

- pcb에서 4축을 한꺼번에 enable disable 하고 있으므로 개별 제어 불가능.

### Homing Start

CAN ID: 0x020

| Byte  | 내용                              |
| ----- | --------------------------------- |
| Byte0 | Motor ID, 0~3. 255면 전체 |
| Byte1 | Homing Mode, 초기에는 0   |

동작:
- board id 구분은 CAN ID를 사용하기 때문에 payload에는 board id를 넣지 않는다.
- Motor ID가 0~3이면 해당 축 homing을 시작한다.
- Motor ID가 255이면 전체 축 homing을 시작한다.
- Homing Mode는 현재 0만 사용한다.
- Enable 상태가 아니거나 ESTOP 상태이면 명령을 무시한다.
- Homing 중 리밋 스위치가 감지되면 해당 축을 정지하고 현재 위치를 0으로 설정한다.


### Clear Error

CAN ID: 0x030

| Byte  | 내용                |
| ----- | ----------------- |
| Byte0 | Motor ID, 255면 전체 |

### Emergency Stop

CAN ID: 0x001

수신 즉시 모든 STEP 출력 정지, 모터 Disable, Queue 초기화, 상태를 ESTOP으로 변경한다.

## STM32 내부 제어 구조

### Queue

CAN RX에서 받은 trajectory point를 내부 링 버퍼 Queue에 저장한다.

```c
typedef struct {
    uint8_t motor_id;
    uint8_t flags;
    int32_t target_pos;
    uint16_t speed;
    uint8_t duration_5ms;
} TrajectoryPoint;
```

Queue 크기는 32개 정도로 한다.

CAN RX 처리에서는 무거운 계산을 하지 말고, 패킷 파싱 후 Queue에 push만 한다.

### Axis 상태 구조체

```c
typedef struct {
    volatile int32_t current_step;
    volatile int32_t target_step;

    int32_t seg_start_step;
    int32_t seg_end_step;
    int32_t seg_delta_step;

    uint16_t seg_total_ms;
    uint16_t seg_elapsed_ms;

    uint8_t moving;
    uint8_t homing;
    uint8_t homing_done;
    uint8_t enabled;
} MotorState;
```

MotorState axis[4]로 관리한다.

## 각도 → step 변환

Raspberry Pi에서 보내는 Target Pos는 기본적으로 0.01도 단위이다.

```c
#define MOTOR_STEPS_PER_REV 200
#define MICROSTEP 16

static const int32_t gear_ratio[4] = {
    20,
    20,
    75,
    30
};
```

변환 공식:

```c
int32_t angle_raw_to_step(uint8_t axis_id, int32_t angle_raw)
{
    int64_t numerator =
        (int64_t)angle_raw *
        gear_ratio[axis_id] *
        MOTOR_STEPS_PER_REV *
        MICROSTEP;

    return (int32_t)(numerator / 36000);
}
```

설명:

* angle_raw = degree * 100
* 360도 = 36000
* 예: 30도 = 3000

Step Mode flag가 1이면 target_pos를 각도값이 아니라 step 수로 직접 해석한다.

## 1ms 타이머 동작

TIM3를 1ms 주기로 설정한다.

1ms 타이머에서는 다음 작업을 수행한다.

1. 현재 segment가 끝났고 Queue에 다음 point가 있으면 pop한다.
2. 새 point를 axis별 segment로 설정한다.
3. 현재 segment가 진행 중이면 선형 보간으로 axis[i].target_step을 갱신한다.

선형 보간 공식:

```c
target_step =
    seg_start_step +
    (seg_delta_step * seg_elapsed_ms) / seg_total_ms;
```

float 사용하지 말고 int32_t / int64_t 기반으로 구현한다.

## 10us 타이머 동작

TIM2를 10us 주기로 설정한다.

10us 타이머에서는 각 축의 current_step이 target_step을 따라가도록 STEP/DIR을 출력한다.

개념:

```c
if (current_step < target_step) {
    DIR = positive;
    STEP pulse 1회;
    current_step++;
}
else if (current_step > target_step) {
    DIR = negative;
    STEP pulse 1회;
    current_step--;
}
```

초기 버전에서는 간단히 10us 인터럽트에서 STEP high 후 짧은 NOP 후 low 처리해도 된다.
나중에 필요하면 high/low 상태머신으로 개선한다.

## Homing

오픈루프 스텝모터이므로 Homing 완료 전에는 위치 명령을 실행하지 않는다.

Homing 절차:

1. Homing Start 명령 수신
2. 해당 축을 home_dir 방향으로 천천히 이동
3. 리밋스위치 Active Low 감지
4. 디바운싱 후 정지
5. current_step = 0
6. target_step = 0
7. homing_done = 1

리밋스위치 디바운싱은 5~10ms 정도로 한다.

## 상태 응답

STM32는 100ms마다 Board1 상태를 CAN ID 0x201로 송신한다.

상태 패킷:

| Byte  | 내용                |
| ----- | ----------------- |
| Byte0 | State             |
| Byte1 | Error Code        |
| Byte2 | Homing Done Bits  |
| Byte3 | Moving Motor ID   |
| Byte4 | Limit Status Bits |
| Byte5 | Queue Free        |
| Byte6 | Enabled Bits      |
| Byte7 | Reserved          |

State 값:

```c
#define STATE_INIT   0
#define STATE_IDLE   1
#define STATE_HOMING 2
#define STATE_MOVING 3
#define STATE_ERROR  4
#define STATE_ESTOP  5
```

## 제약

* HAL 사용 금지
* float 사용 최소화, trajectory와 step 계산은 정수 기반
* CAN RX 인터럽트 처리에서는 무거운 계산 금지
* TMC5160은 부팅 시 초기 설정만 수행
* 이동 중 TMC5160 SPI 설정 변경은 하지 않음
* ODR 방식으로 GPIO 제어
* 빌드 가능한 C 코드로 작성
