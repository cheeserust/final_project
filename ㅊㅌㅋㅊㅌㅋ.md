## 모터 구동 명령 패킷 구조 (RPi → STM32)
Target Pos는 Byte1~4를 합쳐 int32_t로 해석하고, Speed는 Byte5~6을 합쳐 uint16_t로 해석한다. 두 필드는 Little Endian 방식으로 전송한다.


| 바이트        | 필드명                | 자료형           | 단위 / 비트 구조                        | 설명                        |
| ---------- | ------------------ | ------------- | --------------------------------- | ------------------------- |
| **Byte 0** | **Control & ID**   | `uint8_t`     | `[Flags(4bit) \| Motor ID(4bit)]` | 상위 4비트 플래그 + 하위 4비트 모터 ID |
| **Byte 1** | **Target Pos LSB** | `int32_t` 일부  | 0.01도 또는 step                     | 목표 위치 0~7비트               |
| **Byte 2** | **Target Pos**     | `int32_t` 일부  | -                                 | 목표 위치 8~15비트              |
| **Byte 3** | **Target Pos**     | `int32_t` 일부  | -                                 | 목표 위치 16~23비트             |
| **Byte 4** | **Target Pos MSB** | `int32_t` 일부  | -                                 | 목표 위치 24~31비트             |
| **Byte 5** | **Speed LSB**      | `uint16_t` 일부 | 0.01도/s 또는 PPS                    | 목표 속도 0~7비트               |
| **Byte 6** | **Speed MSB**      | `uint16_t` 일부 | -                                 | 목표 속도 8~15비트              |
| **Byte 7** | **Duration**       | `uint8_t`     | 5ms 단위                            | 다음 포인트까지의 이동 시간           |


* **하위 4비트 (Motor ID):** `0` ~ `13` (총 14개 모터 매핑)
* **상위 4비트 (Control Flags):**
* `Bit 4 (Flag 0)`: **Execute** (1: 즉시 실행 )
* `Bit 5 (Flag 1)`: **Relative** (1: 현재 위치 기준 상대 이동)
* `Bit 6 (Flag 2)`: **Step Mode** (1: Target Pos를 step 수로 해석)
* `Bit 7 (Flag 3)`: **Reserved** (예약)



# MoveIt2 → STM32 CAN 제어 구조

MoveIt2에서 생성한 JointTrajectory를 Raspberry Pi에서 CAN 패킷으로 변환해 STM32로 전송합니다.
JointTrajectory point는 우선 50ms 간격으로 전달해 주세요.
STM32에서 1ms 단위로 선형 보간하고, 10µs 타이머로 STEP/DIR을 생성할 예정입니다.


## 전체 흐름

1. MoveIt2가 각 관절의 목표 위치와 시간 정보가 포함된 JointTrajectory를 생성
2. Raspberry Pi가 joint 값을 rad → 0.01도 단위 정수로 변환
3. Raspberry Pi가 8바이트 CAN 패킷으로 패킹 후 STM32로 전송
4. STM32는 수신한 포인트를 Queue에 저장
5. STM32는 1ms 주기로 현재 포인트와 다음 포인트 사이를 선형 보간
6. STM32는 10µs 타이머로 STEP/DIR 펄스를 생성해 모터를 구동

## CAN 패킷

대상 CAN ID:

| CAN ID | 대상 |
|---:|---|
| 0x101 | Board1, 1~4축 |
| 0x102 | Board2, 5축 |
| 0x103 | Board3, 서보보드 |

패킷 구조:

| Byte | 내용 |
|---:|---|
| 0 | 상위 4bit: Flags, 하위 4bit: Motor ID |
| 1~4 | Target Position, int32, 0.01도 단위 |
| 5~6 | Speed, uint16, 0.01도/s 단위 |
| 7 | Duration, 5ms 단위 |

## Motor ID

| Motor ID | 대상 |
|---:|---|
| 0~3 | Board1의 1~4축 |
| 4 | Board2의 5축 |
| 5~12 | Board3의 서보 1~8번 |

## 요청 사항

MoveIt2 쪽에서는 각 joint trajectory point에 대해 다음 값을 넘겨주면 됩니다.

- Joint ID
- Target Position(rad)
- Target Velocity(rad/s)
- 다음 포인트까지의 시간 간격

Raspberry Pi 쪽에서 이를 CAN용 단위로 변환합니다.

## 🚨 2. 시스템 제어 및 비상 정지 패킷 구조 (RPi → 전체)

### 2.1 Emergency Stop (`0x001`) - DLC: 1

모든 보드의 모터 출력을 즉시 차단하고 드라이버를 Disable 합니다.

```text
Byte 0: [ 0x01 ] -> 비상정지 활성화 (나머지 바이트 생략 또는 Reserved)

```

### 2.2 Global Enable / Disable (`0x010`) - DLC: 1

전체 로봇의 모터 토크를 한 번에 켜거나 끕니다.

```text
Byte 0: [ 0x00 ] -> 전체 Disable
Byte 0: [ 0x01 ] -> 전체 Enable

```

## 📈 3. 상태 리포트 패킷 구조 (STM32 → RPi)

* **대상 CAN ID:** `0x201` (Board1), `0x202` (Board2), `0x203` (Board3)
* **패킷 크기:** 8 바이트 (DLC = 8)

| 바이트 | 필드명 | 자료형 | 설명 |
| --- | --- | --- | --- |
| **Byte 0** | **State** | `uint8_t` | 현재 보드 상태 (0:INIT, 1:IDLE, 2:HOMING, 3:MOVING, 4:ERROR, 5:ESTOP) |
| **Byte 1** | **Error Code** | `uint8_t` | 에러 코드 (0:NONE, 1:INVALID_CMD, 2:LIMIT_DETECTED, 3:FAULT) |
| **Byte 2** | **Homing Done** | `uint8_t` | 각 축별 원점복귀 완료 여부 (비트 마스크 형태) |
| **Byte 3** | **Limit Status** | `uint8_t` | 하드웨어 리미트 스위치 감지 상태 비트 |
| **Byte 4** | **Moving Axis** | `uint8_t` | 현재 구동 중인 모터 ID 플래그 |
| **Byte 5** | **Queue Free Space** | `uint8_t` | **[중요] STM32 내부 링 버퍼의 남은 빈 자리 개수 (흐름 제어용)** |
| **Byte 6** | **Reserved** | - | 예약됨 (0 고정) |
| **Byte 7** | **Reserved** | - | 예약됨 (0 고정) |

## 📝 코드 구현 참고용 한 줄 요약 (치트시트)

* **PC 패킹 공식:** `Byte0 = (Flags << 4) | (Motor_ID & 0x0F)`
* **STM32 언패킹 공식:** * `Motor_ID = RxData[0] & 0x0F`
* `Flags = RxData[0] >> 4`
* `Target_Pos = (int32_t)(RxData[1] | (RxData[2]<<8) | (RxData[3]<<16) | (RxData[4]<<24))`
