# Board3 Gripper Control Firmware

## 📌 프로젝트 개요
이 리포지토리는 로봇팔의 3지 9축 그리퍼(Board3)를 제어하기 위한 펌웨어 코드를 포함하고 있습니다. 중앙 서버(또는 RPi)와 CAN 통신으로 명령을 주고받으며, 내부에 I2C 통신 설정이 포함되어 있고, 하프 듀플렉스(Half-Duplex) UART를 통해 9개의 Feetech SCS0009 서보 모터를 제어합니다. 레지스터 직접 제어(Bare-metal) 방식을 채택하여 지연 시간을 최소화하고 정밀한 하드웨어 제어를 구현했습니다.

## 📡 시스템 및 통신 아키텍처
* **CAN (SPI2 - MCP2515):** 서버와 Board3 간의 메인 통신을 담당합니다. 0x103(명령 수신), 0x203(상태 보고), 0x303(고속 위치 피드백) 등의 ID를 사용하여 데이터를 송수신합니다.
* **UART (USART1 - Half-Duplex):** 1Mbps 속도로 서보 모터와 통신합니다. 하나의 데이터 선으로 송수신을 처리하기 위해 송신(TX)과 수신(RX) 레지스터 모드를 동적으로 스위칭하여 에코 현상을 방지합니다.
* **I2C (SC16IS752):** 보드 내부의 추가적인 확장 포트 제어 및 센서 통신을 위해 구성되어 있습니다.

## 🧠 핵심 알고리즘 및 제어 로직

### 1. CAN 9-Frame Staging 알고리즘
CAN 통신 대역폭의 한계로 인해 모터 1개당 1개의 프레임이 분할되어 전송됩니다. 펌웨어는 수신된 프레임을 즉시 실행하지 않고, **내부 버퍼에 9개의 모터(ID 0~8) 명령이 모두 도착할 때까지 대기(Staging)**합니다.
* 타임아웃(100ms) 내에 모든 프레임이 도착하고, 9개 명령의 구동 시간(Duration)이 완벽히 일치할 때만 유효한 명령(`g_cmd.is_new_cmd = 1`)으로 인정하여 오작동과 데이터 꼬임을 원천 차단합니다.

### 2. 논블로킹(Non-blocking) 부하 감지 및 백오프(Backoff)
그리퍼가 물체를 잡을 때 발생하는 물리적 충격과 서보 모터 과전류를 방지하기 위한 유한 상태 기계(State Machine)가 적용되어 있습니다.
* **부하 감지:** 설정된 임계값(Threshold) 이상의 부하가 감지되면 해당 모터의 이동을 즉시 중단합니다.
* **논블로킹 타이머:** `g_ms_tick`을 이용해 메인 CPU의 루프를 멈추지 않고(Delay 없이) 3ms 동안 안전 대기합니다.
* **후퇴 방어 기제:** 대기 후 부하를 받은 반대 방향으로 5 스텝 물러나(50ms 동안 부드럽게) 모터 기어에 가해지는 장력을 풀어주고 `CONTACT_HOLD` 상태로 안전하게 진입합니다.

### 3. Sync Write 다중 제어 및 고속 위치 피드백
* **동기화 송신:** 9개의 모터가 동시에 움직이도록 Feetech 프로토콜의 `Sync Write` 명령(0x83)을 사용하여 9개의 제어 패킷을 하나로 묶어 송신 지연을 최소화했습니다.
* **위치 피드백 분산:** MoveIt2와의 동기화를 위해 20ms 주기마다 0x303 CAN ID를 통해 현재 위치를 서버로 보고합니다. 9개의 모터 데이터를 한 번에 보내면 CAN 버스 병목이 생기므로, 3개씩 그룹화(인덱스 0x01, 0x02, 0x03)하여 분할 전송합니다.

### 4. 직관적인 0.01도 스케일 맵핑
중앙 서버는 기구적 구조를 신경 쓰지 않고 직관적인 각도(0.01도 단위, 예: 3000 = 30.00도)로 명령을 내립니다. 펌웨어 내부에서 각 손가락 모터의 조립 방향(Direction 배열)과 고유 영점(Home Goal)을 반영하여 SCS0009 모터의 실제 하드웨어 스텝(0~1023)으로 안전하게 변환(`Convert_Angle_To_Step`) 및 클램핑 처리합니다.

## 📂 주요 파일 구성

| 파일명 | 주요 역할 |
| :--- | :--- |
| `main.c` | 메인 루프 실행, 1ms 타이머 생성 및 CAN 통신(MCP2515) 프레임 수신/분석 |
| `gripper_control.c` | 모터 제어 태스크, 상태 머신 관리, 논블로킹 백오프 알고리즘 구현 |
| `feetech.c / .h` | 서보 모터 프로토콜 구현, 체크섬 계산, Sync Write 및 위치/부하 읽기 |
| `uart.c` | Half-Duplex 기반 USART1 제어, 타임아웃 기반 수신 처리 |
| `gripper_shared.h` | CAN 통신부와 제어부가 함께 공유하는 칠판(구조체) 및 상태 코드 정의 |
| `gripper_mapping.h` | 모터별 방향 보정 배열 및 각도 ↔ 스텝 변환 인라인 함수 포함 |
